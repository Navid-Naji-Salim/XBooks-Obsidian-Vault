# Customers

The **Customers** page is accessed through the **Sales** section of the application.

Unlike simpler navigation items, the **Sales** tab is designed as a dropdown menu that contains multiple sub-pages. One of these sub-pages is **Customers**. Additional sales-related pages will be introduced later as the application grows.

## Page Layout

The Customers page is intentionally designed to follow the same layout philosophy as the **Items** page in order to maintain a consistent user experience throughout the application.

The page is divided into two primary sections:

- **Top Section** — Customer creation form.
- **Bottom Section** — Customer list, where all existing customers can be viewed, searched, and scrolled through.

This familiar layout allows users to quickly understand how different parts of the application work while keeping the interface clean and predictable.

## Design Philosophy

The Customers page takes inspiration from **Zoho Books**, but it is **not intended to be a replica**.

The goal is to adopt proven UX patterns while making deliberate design decisions that better suit the scope of this project. Features that provide genuine value to a basic-to-moderate accounting application will be kept, while enterprise-specific functionality that unnecessarily increases complexity will be omitted.

The exact fields and functionality included in the customer creation form will be determined by evaluating the relevant Zoho Books customer creation screens and selecting only the features that meaningfully contribute to the application's goals.
